# `_wmbench_src/` provenance

This directory is a **frozen, hard copy** of selected files from the wmbench
repository, ingested for use by `tools/build_snapshot.py`. The phyground.github.io
build does not require the wmbench checkout to exist after Round 1.

## Score JSON slimming

The original `data/scores/<evaluator>/.../<id>.json` files in wmbench carry per-prompt
debug fields (`raw_response_general`, `raw_response_physical`, `rationale_general`,
`rationale_physical`, `usage`) totalling **~10 KB per result × 250 results × 74 files
≈ 90 MB**. We strip those five fields at copy time to keep the public repo small.

- Original total bytes: **89,578,123** (90 MB)
- Slimmed total bytes:  **11,629,614** (12 MB; 13.0% of original)
- Public-relevant fields preserved: `meta`, `num_videos`, `general_dimensions`,
  `judge`, `visual_config`, `inference_config`, `prompt_config`, `prompt_mode`, and
  `results[*]` minus the five debug fields above (so per-result fields kept are
  `SA`, `PTV`, `persistence`, `general_avg`, `physical`, `video`, `prompt`,
  `physical_laws`).
- Slimming is documented in `tools/build_snapshot.py` so a downstream re-ingest can
  reproduce the same shape.

If a future round needs the raw rationales (e.g. for a "why was this scored 2/5?"
detail panel), the canonical originals stay in the wmbench repo and can be re-fetched.

## HF asset staging

Video bytes for the public dataset live **outside** git (see
`_wmbench_src/data/videos/` and `_wmbench_src/data/paperdemo/**/*.mp4` in
`.gitignore`):

- `tools/stage_hf_assets.py /path/to/wmbench` walks
  `snapshot/HF_UPLOAD_MANIFEST.json` and copies every missing video /
  first-frame source from the supplied wmbench checkout into the
  corresponding `_wmbench_src/data/...` path. Idempotent on re-run.
- Round-9 scope: humaneval-100 prompts × the 8 strict-intersection
  models = ~800 videos, plus ~83 first-frames. Staged tree ≈ 800 MB
  (gitignored).
- The build emits HF URLs only for assets that are present on the
  `juyil/phygroundwebsitevideo` dataset (8 model dirs + flat
  `first_images/`). Manifest-only fallback models (`cogvideox1.5-5b-i2v`,
  `hunyuanvideo-i2v`, `ltx-2-19b-distilled-fp8`) are dropped from
  `per_model_videos`; `tools/build_snapshot.py:_HF_PUBLISHED_MODELS`
  pins the allowlist.
- `snapshot/HF_UPLOAD_MANIFEST.json` reports `n_missing_locally == 0`
  after staging — every embedded HF URL has a local source byte.

### Historical note (Round 7, superseded)

Earlier rounds emitted URLs for all 250 humaneval prompts × the 4-model
prompt-manifest subset and used a `paperdemo/<law>/<file>.mp4` HF path
that did not exist on the published dataset. Round 9 narrowed the scope
to the 100-prompt published set and remapped paperdemo videos to
`videos/<model>/<stem>.mp4` to match the dataset layout.

## Refresh procedure

1. Re-run the byte-exact copy of the verbatim files and the slimming step for
   `data/scores/`. The slimming policy is in `tools/build_snapshot.py`.
2. Update this file's "Score JSON slimming" totals if the upstream sizes change.
3. `python tools/build_snapshot.py --select-humaneval-100`
4. `python tools/verify_snapshot.py`
5. `python tools/build_site.py --config snapshot/index/site_config.json`
6. Commit.
