repository, ingested for use by `tools/build_snapshot.py`. The phyground.github.io
build does not require the wmbench checkout to exist after Round 1.

## Score JSON slimming

The original `data/scores/<evaluator>/.../<id>.json` files in wmbench carry per-prompt
debug fields (`raw_response_general`, `raw_response_physical`, `rationale_general`,
`rationale_physical`, `usage`) totalling **~10 KB per result Г— 250 results Г— 74 files
в‰€ 90 MB**. We strip those five fields at copy time to keep the public repo small.

- Original total bytes: **89,578,123** (90 MB)
- Slimmed total bytes:  **11,629,614** (12 MB; 13.0% of original)
- Public-relevant fields preserved: `meta`, `num_videos`, `general_dimensions`,
  `judge`, `visual_config`, `inference_config`, `prompt_config`, `prompt_mode`, and
  `results[*]` minus the five debug fields above (so per-result fields kept are
  `SA`, `PTV`, `persistence`, `general_avg`, `physical`, `video`, `prompt`,
  `physical_laws`).
- Slimming is documented in `tools/build_snapshot.py` so a downstream re-ingest can
  reproduce the same shape.

If a future round needs the raw rationales (e.g. for a "why was this scored 2/5?"
detail panel), the canonical originals stay in the wmbench repo and can be re-fetched.

## Refresh procedure

1. Re-run the byte-exact copy of the verbatim files and the slimming step for
   `data/scores/`. The slimming policy is in `tools/build_snapshot.py`.
2. Update this file's "Score JSON slimming" totals if the upstream sizes change.
3. `python tools/build_snapshot.py --select-humaneval-100`
4. `python tools/verify_snapshot.py`
5. `python tools/build_site.py --config snapshot/index/site_config.json`
6. Commit.
